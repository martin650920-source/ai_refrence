# Claude Config 統一管理架構設計

> 目標：將散落在 Windows、WSL、Claude Code CLI、claude.ai 的 skill/rule/mcp/context 統一管理，
> 以 Google Drive 為 Single Source of Truth，透過 symlink 與一個 `context-loader` skill 自動載入。

---

## 問題現況

| Claude 入口 | config 散落位置 |
|---|---|
| claude.ai web | `/mnt/skills/public/`（平台管理，唯讀） |
| Claude Code CLI (WSL) | `~/.claude/CLAUDE.md`、`~/.claude/mcp.json` |
| VS Code Remote SSH | 同 CLI，但 cwd 依專案不同 |
| Windows Claude Desktop | `%APPDATA%\Claude\` |

**散落檔案清單：**
- `CLAUDE.md`（全域 + 專案各一份）
- `.mcp.json` / `mcp_server_config`
- 自訂 `SKILL.md`
- `docs/test-strategy.md`、Robot Framework resource
- Ceedling config、Nagra/TNTSAT context

---

## 目標架構

```
Google Drive (Single Source of Truth)
└── claude-config/
    ├── skills/
    │   ├── context-loader/
    │   │   └── SKILL.md          # 自動載入 skill（見下方設計）
    │   └── <其他自訂 skill>/
    ├── rules/
    │   ├── global.md             # 所有專案共用規則
    │   └── sym6-tntsat.md        # 專案專屬規則
    ├── mcp/
    │   └── mcp_template.json     # MCP server config 範本
    └── context/
        ├── global.md             # 常駐全域 context
        ├── nagra-tntsat.md       # Nagra/TNTSAT 背景知識
        └── test-strategy.md      # 同步自 docs/test-strategy.md
```

## 實作步驟

### Step 1：Windows 側

1. 安裝 **Google Drive for Desktop**（若尚未安裝）
2. 確認同步路徑，例如：
   ```
   C:\Users\<you>\Google Drive\My Drive\claude-config\
   ```
3. 建立上述目錄結構，把現有散落檔搬入

### Step 2：WSL 側建立 symlink

```bash
# Google Drive 在 WSL 的掛載路徑（依實際調整）
GDRIVE="/mnt/c/Users/<you>/Google Drive/My Drive/claude-config"

# 將自訂 skills 連結到 Claude Code 能讀到的位置
mkdir -p ~/.claude/skills
ln -sfn "$GDRIVE/skills" ~/.claude/skills/gdrive

# 全域 CLAUDE.md 從 GDrive 管理（備份舊的再換）
cp ~/.claude/CLAUDE.md "$GDRIVE/rules/claude-md-backup.md"
ln -sfn "$GDRIVE/rules/global.md" ~/.claude/CLAUDE.md

# context 資料夾連結（供 Claude Code Read 指令使用）
ln -sfn "$GDRIVE/context" ~/claude-context
```

### Step 3：各專案 CLAUDE.md 加入 context-loader 引用

在每個專案根目錄的 `CLAUDE.md` 開頭加：

```markdown
## Bootstrap
At the start of every new session, read `~/.claude/skills/gdrive/context-loader/SKILL.md`
and follow its instructions to load the appropriate context for this project.
```

### Step 4：MCP config 管理

```bash
# WSL Claude Code 的 MCP config 改用 GDrive 版本
ln -sfn "$GDRIVE/mcp/mcp_template.json" ~/.claude/mcp.json
```

## context-loader Skill 設計

**檔案位置：** `claude-config/skills/context-loader/SKILL.md`

**邏輯流程：**

```
開始新 session
    │
    ▼
偵測 cwd 特徵檔
    ├── CMakeLists.txt + mt_unf_xxx → [sym6 porting layer]
    ├── robot/ + *.robot            → [HIL acceptance test]
    ├── Ceedling (project.yml)      → [unit test]
    └── 無特徵                      → [通用]
    │
    ▼
依類型 Read 對應 context
    ├── sym6:  ~/claude-context/nagra-tntsat.md
    │          ~/claude-context/test-strategy.md
    ├── robot: robot/resources/*.robot (列出檔案讓我選)
    ├── unit:  ~/claude-context/test-strategy.md
    └── 通用:  ~/claude-context/global.md
    │
    ▼
列出已載入清單，等待確認後繼續
```

**SKILL.md 草稿內容：**

```markdown
---
name: context-loader
description: >
  At the start of a Claude Code session, auto-detect the project type from cwd
  and load the appropriate context files. Trigger whenever CLAUDE.md says to
  read this skill, or when the user says "load context" or "bootstrap session".
---

# Context Loader

## Step 1: Detect Project Type

Check for these marker files in cwd:

| Marker | Project Type |
|---|---|
| `CMakeLists.txt` AND any `mt_unf_*.h` in include/ | `sym6-porting` |
| `robot/` directory AND `*.robot` files | `hil-robot` |
| `project.yml` (Ceedling) | `unit-test` |
| none of the above | `generic` |

## Step 2: Load Context by Type

### sym6-porting
Read these files in order:
1. `~/claude-context/nagra-tntsat.md`
2. `~/claude-context/test-strategy.md`
3. `include/` — list headers, ask which modules are in scope today

### hil-robot
Read these files in order:
1. `~/claude-context/nagra-tntsat.md`
2. `robot/resources/` — list all `.robot` and `.resource` files, ask which to load

### unit-test
Read these files in order:
1. `~/claude-context/test-strategy.md`
2. `test/` — list test suites, ask which are in scope

### generic
1. `~/claude-context/global.md`

## Step 3: Confirm and Proceed

Output a summary:
```
## Session Context Loaded
- Project type: <type>
- Files loaded:
  - <file1>
  - <file2>
Ready. What are we working on today?
```
```

## 替代方案（簡列）

| 方案 | 優點 | 缺點 |
|---|---|---|
| **Git private repo + clone** | 版本控制完整 | 需手動 pull，即時性差 |
| **rclone + cron** | 不依賴 GDrive Desktop | 需排程設定，WSL headless 可用 |
| **dotfiles + stow** | 標準 Unix 做法 | 不含 Windows 同步 |
| **Obsidian Vault (GDrive)** | 可視化編輯 context | 多一層工具依賴 |

---

## 建議執行順序

1. **今天**：建立 GDrive `claude-config/` 目錄，把現有檔案搬進去
2. **今天**：在 WSL 建立 symlink，驗證 Claude Code 能讀到
3. **之後**：把 context-loader SKILL.md 草稿貼給 Claude Code，讓它幫你完成細節並測試
4. **之後**：把 context-loader 加進每個專案的 CLAUDE.md bootstrap 區塊
